# ERP系统用户使用手册

## 第一章 系统概述

### 1.1 系统简介

本ERP系统是一套功能完善的企业资源计划管理系统。
涵盖了零售管理、采购管理、销售管理、仓库管理、财务管理、报表查询、商品管理、基础资料和系统管理等核心业务模块。

系统采用前后端分离架构。
后端基于Spring Boot框架构建。
前端采用Vue.js技术栈。
支持多租户模式，适用于中小型企业的日常运营管理。

系统的核心价值在于帮助企业实现业务流程的数字化管理。
通过规范化的数据录入和流程控制，提高企业运营效率，降低管理成本。

系统支持多角色权限管理。
不同角色的用户可以看到和操作不同的功能模块。
确保数据安全和业务规范。

系统提供了直观的图形化界面。
用户无需专业培训即可快速上手使用。
大大降低了系统推广的难度和成本。

系统的技术特点包括多个方面：

- 采用HTTPS加密传输协议，有效保障数据传输过程中的安全性。
  防止敏感数据被窃取或篡改。

- 基于Redis实现会话管理，支持分布式部署和负载均衡。
  能够应对高并发访问场景。

- 采用MyBatis-Plus作为持久层框架。
  通过对象关系映射简化了数据库操作，降低了开发难度。

- 集成了Swagger接口文档。
  开发者和管理员可以方便地了解和使用各个API接口。
  - 默认访问地址：http://localhost:9999/swagger-ui.html
  - 增强版UI地址：http://localhost:9999/doc.html

### 1.2 系统功能模块

系统按照企业日常运营管理的实际需求，将功能划分为多个核心模块。
功能模块按照系统中的展示顺序排列如下：

**零售管理**

零售管理模块用于处理企业的零售业务。
包括零售出库和零售退货等功能。

该模块支持快速开单、扫码录入等便捷操作。
适合超市、便利店等零售场景使用。

**采购管理**

采购管理模块负责企业的采购业务流程。
包括请购单、采购订单、采购入库和采购退货等功能。

通过该模块，企业可以规范采购流程。
实现从需求提出到货物入库的全流程管理。

**销售管理**

销售管理模块负责企业的销售业务流程。
包括销售订单、销售出库和销售退货等功能。

该模块与客户信息、库存管理等模块紧密关联。
帮助企业实现销售业务的规范化管理。

**仓库管理**

仓库管理模块用于管理企业的库存业务。
包括其他入库、其他出库、调拨出库、组装单和拆卸单等功能。

通过该模块，企业可以实时掌握库存动态。
优化库存结构，降低库存成本。

**财务管理**

财务管理模块负责企业的资金管理和财务核算。
包括收入单、支出单、收款单、付款单、转账单和收预付款等功能。

该模块与采购、销售等业务模块紧密联动。
自动生成财务记录，提高财务工作效率。

**报表查询**

报表查询模块为企业管理层提供决策支持。
包括商品库存、账户统计、零售统计、采购统计、销售统计、入库明细、出库明细、调拨明细、入库汇总、出库汇总、进销存统计、客户对账、供应商对账和库存预警等功能。

用户可以根据自身需求选择不同的报表。
获取所需的业务数据和统计分析结果。

**商品管理**

商品管理模块是企业资源管理的基础。
包括商品类别、商品信息、多单位和多属性等功能。

通过建立完善的商品档案和分类体系。
帮助企业准确掌握商品信息，为其他业务模块提供数据支持。

**基础资料**

基础资料模块用于维护系统的基础数据。
包括供应商信息、客户信息、会员信息、仓库信息、收支项目、结算账户和经手人管理等功能。

这些基础数据是系统正常运行的保障。
需要在系统初始化阶段完成设置。

**系统管理**

系统管理模块用于维护系统的正常运行。
包括角色管理、用户管理、机构管理、日志管理、商品属性、系统配置、功能管理、租户管理、插件管理和平台配置等功能。

该模块通常仅对系统管理员开放。
用于系统的日常维护和管理。

### 1.3 系统架构

系统采用经典的三层架构设计：
- 表现层
- 业务逻辑层
- 数据访问层

三个层次相互独立又紧密协作。

表现层负责用户界面交互。
通过RESTful API与后端进行数据通信。
采用前后端分离的开发模式，前端代码独立部署，互不干扰。

业务逻辑层处理核心业务规则和数据验证。
是系统的核心所在，所有业务逻辑都在这一层实现。

数据访问层通过MyBatis-Plus框架与MySQL数据库进行交互。
采用面向对象的方式操作数据库，大大简化了数据持久化操作的复杂度。

系统支持多租户隔离。
每个租户拥有独立的数据空间。
通过tenantId字段进行数据隔离。

这种设计使得系统能够同时为多个企业提供服务。
每个企业的数据都相互独立、安全可靠。

系统管理员可以管理所有租户的数据。
普通用户只能访问其所属租户的数据。
确保了数据的安全性和隔离性。

系统的部署架构支持水平扩展。
可以通过增加服务实例来应对更高的并发访问需求。

前端静态资源通过CDN分发，提高页面加载速度。
后端服务支持集群部署，通过负载均衡器分发请求。
实现请求的负载均衡和故障转移，提高系统的可用性和稳定性。

### 1.4 运行环境要求

后端服务需要Java 8或更高版本运行环境。
建议内存配置不低于4GB，磁盘空间不低于50GB。

数据库采用MySQL 5.7或MySQL 8.0。
需要预先创建数据库并配置相应的连接信息： //告诉用户在什么文件改什么语句
- 数据库名称
- 用户名
- 密码等

Redis缓存服务用于存储会话信息和热点数据。
建议配置为主从复制模式以保证高可用性。
同时需要设置合理的过期时间和内存限制。

前端服务需要Node.js 14.x或更高版本。
通过npm或yarn包管理器安装依赖。

开发环境下可以直接运行npm run dev命令启动开发服务器。
生产环境建议使用Nginx作为Web服务器。
配置HTTPS证书和静态资源缓存策略。
以提高页面加载速度和安全性。

系统默认配置下：
- 后端服务监听8443端口（HTTPS）和8080端口（HTTP）
- 前端服务默认运行在3000端口

请确保这些端口在防火墙中已开放，并且没有其他服务占用。

如果需要修改端口配置，可以修改application.yml配置文件中的相应配置项。

## 第二章 系统登录

### 2.1 访问系统

打开Chrome、Firefox或Edge等现代浏览器。
在地址栏输入系统的前端访问地址，进入系统登录页面。

建议使用最新版本的浏览器访问系统。
以获得最佳的用户体验和安全性保障。

如果页面无法正常显示，请按以下步骤排查问题：
1. 确认网络连接正常，能够访问服务器地址
2. 检查浏览器是否支持JavaScript，建议启用Cookie和本地存储功能
3. 确认系统服务已经启动运行，可以通过直接访问后端API地址来验证服务是否正常

登录页面采用简洁的设计风格。
页面中央为登录表单，包含用户名输入框、密码输入框和登录按钮。

页面顶部显示系统名称和版本信息。
底部显示技术支持联系方式和版权声明。

页面背景采用渐变色设计，视觉体验舒适大方。

### 2.2 用户登录

在登录页面，按照以下步骤完成登录操作：

**第一步：在用户名输入框中输入您的登录账号。**

系统默认的管理员账号为"jsh"。
普通用户账号由系统管理员分配。

如果忘记用户名，请联系系统管理员获取。
用户名不支持特殊字符，长度限制为3-20个字符。

**第二步：在密码输入框中输入与账号对应的密码。**

系统默认密码为"123456"。
首次登录后建议立即修改为更安全的密码。

密码输入框支持显示隐藏切换。
点击输入框右侧的眼睛图标可以切换密码的显示状态。
方便用户确认输入是否正确。

**第三步：点击"登录"按钮提交登录请求。**

系统会验证用户名和密码的正确性。
验证通过后将自动跳转到系统首页。

如果用户名或密码错误，页面会显示相应的错误提示信息。
提示用户重新输入。

### 2.3 登录失败处理

如果登录失败，页面会显示具体的错误信息。
常见的错误信息及处理方法如下：

**错误信息：用户不存在。**
- 请检查用户名输入是否正确，注意区分大小写
- 如果确认用户名正确但仍然提示用户不存在，可能是账号已被禁用或删除
- 请联系系统管理员确认账号状态
- 系统默认尝试登录次数为5次，超过限制后账号会被临时锁定15分钟

**错误信息：密码错误。**
- 请确认密码输入是否正确，注意检查大小写和特殊字符
- 如果多次密码错误，账号可能会被临时锁定
- 请等待15分钟后重试或联系系统管理员解锁
- 系统会记录每次登录失败的IP地址和账号信息，用于安全审计

**错误信息：验证码错误。**
- 如果系统启用了验证码功能，需要正确输入图片中显示的验证码字符
- 验证码不区分大小写，但需要注意区分外观相似的字符
  如数字0和字母O、数字1和字母l、数字8和字母B等
- 如果看不清验证码，可以点击验证码图片刷新重新获取

**错误信息：账号已过期。**
- 如果系统设置了账号有效期，账号过期后将无法登录
- 请联系系统管理员续期或重新开通账号
- 企业版用户可以联系技术支持进行处理

### 2.4 修改初始密码

首次登录系统后，建议立即修改初始密码，以提高账户安全性。

修改密码的步骤如下：

**第一步：点击页面右上角的用户名，在下拉菜单中选择"个人设置"选项。**

点击用户名后会展开一个下拉菜单。
菜单中包含"个人设置"、"消息通知"和"退出登录"三个选项。

**第二步：在个人设置页面中，找到"修改密码"模块。**

页面会显示当前用户的基本信息：
- 用户名
- 姓名
- 邮箱
- 手机号等

在页面下方找到"修改密码"区域。
分别输入当前密码、新密码和确认密码。

**第三步：设置新密码。**

新密码需要满足以下复杂度要求：
- 长度不少于8个字符
- 包含大写字母、小写字母、数字和特殊字符中的至少三种类型
- 新密码不能与最近使用过的5个密码相同

| 密码要求 | 说明 |
|---------|------|
| 最小长度 | 8个字符 |
| 字符类型 | 至少包含大写字母、小写字母、数字中的两种 |
| 特殊字符 | 建议包含至少一个特殊字符 |
| 历史检查 | 不能与最近5个密码相同 |

**第四步：点击"保存"按钮提交修改请求。**

系统会验证当前密码的正确性。
并检查新密码的复杂度是否符合要求。

修改成功后，系统会提示密码修改成功。
下次登录时需要使用新密码。

## 第三章 系统主界面

### 3.1 界面布局概览

成功登录系统后，将进入系统主界面。

主界面采用标准的Web应用布局设计。
由顶部导航栏、左侧菜单栏和主内容区域三部分组成。

这种布局方式充分利用了屏幕空间。
同时保持了界面的简洁和美观。

**顶部导航栏**位于页面最上方。
包含系统logo、当前用户信息、功能菜单和退出登录按钮。

导航栏高度为60像素。
采用固定定位方式，始终显示在页面顶部。
即使页面内容较长也不会消失。

**左侧菜单栏**采用折叠式设计。
默认显示一级功能菜单。
点击菜单项可以展开显示二级功能。

菜单栏宽度为200像素。
支持拖拽调整宽度，可以根据实际需要展开或折叠。

**主内容区域**占据页面的大部分空间。
用于显示具体的功能操作界面。

内容区域采用响应式设计。
能够根据浏览器窗口大小自动调整布局。
确保在各种分辨率下都能获得良好的显示效果。

### 3.2 顶部导航栏

顶部导航栏提供了系统级的功能入口和用户信息展示。
从左到右依次包含以下元素：

**系统Logo区域**：
- 显示系统名称和图标，点击可以返回首页
- Logo旁边显示当前系统的版本号，方便用户了解系统版本信息
- Logo区域支持点击，点击后系统会自动跳转到首页

**搜索功能区**（部分版本支持）：
- 提供全局搜索功能
- 可以快速查找商品、客户、供应商等业务数据
- 输入关键字后系统会自动显示匹配的搜索结果
- 支持按类型筛选

**用户信息区域**：
- 显示当前登录用户的头像（如果有）和用户名
- 点击用户名会展开一个下拉菜单，包含三个选项：
  - "个人设置"
  - "消息通知"
  - "退出登录"

**个人设置**：
- 点击进入个人设置页面
- 可以修改个人信息、密码等设置
- 个人设置页面还显示用户的登录历史记录
- 方便用户了解账号使用情况

**消息通知**：
- 点击查看系统消息和通知
- 包括待办事项提醒、系统公告、业务审批通知等
- 消息数量以红色数字角标的形式显示在消息图标右上角
- 点击后可以查看全部消息并对消息进行已读处理

**退出登录**：
- 点击退出当前账号，返回登录页面
- 退出时系统会清除会话信息，确保账号安全
- 建议在离开电脑前主动退出登录，特别是公共场合使用时

### 3.3 左侧菜单栏

左侧菜单栏是系统功能导航的主要入口。
采用树形结构组织功能模块。

菜单栏支持折叠和展开两种状态。
可以通过点击顶部的手风琴图标或菜单栏边缘的折叠按钮进行切换。

菜单栏包含以下一级功能模块，按照系统实际展示顺序排列：

| 模块名称 | 功能描述 |
|---------|---------|
| 零售管理 | 处理企业零售业务，包括零售出库和零售退货 |
| 采购管理 | 管理采购业务流程，包括请购单、采购订单、采购入库和采购退货 |
| 销售管理 | 管理销售业务流程，包括销售订单、销售出库和销售退货 |
| 仓库管理 | 管理库存业务，包括其他入库、其他出库、调拨出库、组装单和拆卸单 |
| 财务管理 | 负责资金管理和财务核算，包括收入单、支出单、收款单、付款单等 |
| 报表查询 | 提供各类业务报表，包括商品库存、采购统计、销售统计等 |
| 商品管理 | 管理商品信息，包括商品类别、商品信息、多单位和多属性 |
| 基础资料 | 维护系统基础数据，包括供应商、客户、仓库等信息 |
| 系统管理 | 维护系统正常运行，包括用户管理、角色管理、系统配置等 |

每个一级菜单项前面有一个箭头图标。
点击箭头可以展开显示该模块下的二级功能菜单。

点击具体的菜单项，系统会在主内容区域加载相应的功能页面。

当前选中的菜单项会以高亮显示。
方便用户了解当前所在位置。

### 3.4 主内容区域

主内容区域是用户进行具体业务操作的主要工作区域。

不同功能页面有不同的布局和操作方式。
但通常包含以下通用元素：

**页面标题**：
- 显示当前页面的名称，通常位于内容区域顶部
- 页面标题下方通常包含面包屑导航
- 显示当前页面在系统中的位置层级

**工具栏**：
- 包含页面级的操作按钮，如新增、导入、导出、刷新等
- 工具栏通常位于页面标题下方
- 按钮按功能分组排列
- 将鼠标悬停在按钮上会显示该按钮的功能说明

**数据表格**：
- 以表格形式展示业务数据
- 支持排序、筛选、分页等操作
- 表格的列标题可以点击进行升序或降序排列
- 列标题右侧的筛选图标可以打开列筛选器
- 表格支持行选择，选中行后会高亮显示
- 支持单选和多选

**分页控件**：
- 位于数据表格下方，用于控制数据的分页显示
- 分页控件包含页码导航、每页显示条数设置、跳转到指定页等功能
- 系统默认每页显示10条数据
- 用户可以根据需要调整为10、20、50或100条

## 第四章 零售管理

### 4.1 功能概述

零售管理模块用于处理企业的零售业务。
是企业直接面向终端客户的销售渠道管理模块。

该模块支持快速开单、扫码录入等便捷操作。
适合超市、便利店等零售场景使用。

### 4.2 零售出库

零售出库用于记录企业的零售销售业务。
当企业向终端客户销售商品时，需要创建零售出库单。

**操作步骤：**

1. 在左侧菜单栏中依次点击"零售管理" → "零售出库"
2. 进入零售出库单列表页面
3. 点击工具栏中的"新增"按钮，弹出新增零售出库单对话框
4. 填写零售出库单信息，包括客户信息、商品信息、数量、单价等
5. 确认信息无误后，点击"保存"按钮

### 4.3 零售退货

零售退货用于处理客户退货业务。
当客户需要退回已购买的商品时，需要创建零售退货单。

**操作步骤：**

1. 在左侧菜单栏中依次点击"零售管理" → "零售退货"

3. 点击工具栏中的"新增"按钮，弹出新增零售退货单对话框
4. 填写零售退货单信息，包括客户信息、商品信息、数量、单价等

## 第十三章 常见问题与解决方案

### 13.1 登录相关问题

**问题一：忘记密码怎么办？**

如果忘记了登录密码，可以通过以下方式重置密码：
1. 联系系统管理员，说明情况并申请重置密码
2. 管理员登录系统后，在用户管理中找到对应用户
3. 点击"重置密码"按钮，将密码重置为初始密码
4. 重置后首次登录时，系统会强制要求修改密码

**问题二：账号被锁定怎么办？**

系统默认在连续5次密码错误后锁定账号15分钟。
锁定期间无法登录，可以采取以下措施：
1. 等待15分钟后自动解锁
2. 联系系统管理员立即解锁
3. 如果多次被锁定，建议检查是否账号信息泄露

### 13.2 业务操作问题

**问题三：创建单据时找不到商品怎么办？**

如果在创建单据时无法找到需要的商品，请按以下步骤排查：
1. 确认商品是否已经创建：在"商品管理"中搜索商品名称或编号
2. 确认商品是否关联了正确的分类：商品需要归属于某个商品分类才能被检索
3. 确认用户是否有查看该商品的权限：联系管理员检查角色权限设置
4. 确认商品是否被标记为停用：停用的商品不会显示在选择列表中

**问题四：单据审核不通过怎么办？**

如果提交的单据被审核人员驳回，请按以下步骤处理：
1. 在单据列表中找到被驳回的单据，查看驳回原因
2. 点击单据进入详情页面，修改单据内容
3. 修改完成后重新提交审核
4. 如有疑问，可以联系审核人员沟通确认

### 13.3 系统技术问题

**问题五：页面加载缓慢怎么办？**

如果系统页面加载速度较慢，可以尝试以下优化措施：
1. 检查网络连接是否稳定
2. 清除浏览器缓存和Cookie
3. 尝试使用其他浏览器（如Chrome、Firefox）
4. 减少同时打开的浏览器标签页数量
5. 联系管理员检查服务器性能和网络状况

**问题六：数据导出失败怎么办？**

如果导出报表时出现失败，请按以下步骤排查：
1. 检查导出数据量是否过大：大量数据导出可能需要较长时间
2. 确认浏览器是否允许下载文件：检查浏览器的弹出窗口拦截设置
3. 尝试分批次导出：将数据分成小批次分别导出
4. 更换浏览器或电脑后重试
