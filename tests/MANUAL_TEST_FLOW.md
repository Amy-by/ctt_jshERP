# jshERP系统手动测试详细流程

## 1. 系统登录测试

### 1.1 正常登录流程

**测试步骤**：
1. 打开浏览器，访问前端地址：http://localhost
2. 在登录页面，输入以下信息：
   - 租户账号：jsh
   - 用户名：admin
   - 密码：123456
3. 点击「登录」按钮

**字段说明**：
- 租户账号：系统多租户标识，必填，固定值为jsh
- 用户名：登录用户账号，必填
- 密码：登录用户密码，必填

**预期结果**：
- 成功登录系统，进入首页
- 显示系统概览和导航菜单

### 1.2 异常登录场景

**测试场景1：空租户账号**
- 操作：租户账号留空，输入用户名和密码
- 预期结果：提示「租户账号不能为空」

**测试场景2：空用户名**
- 操作：用户名留空，输入租户账号和密码
- 预期结果：提示「用户名不能为空」

**测试场景3：空密码**
- 操作：密码留空，输入租户账号和用户名
- 预期结果：提示「密码不能为空」

**测试场景4：错误租户账号**
- 操作：输入错误的租户账号，正确的用户名和密码
- 预期结果：提示「租户不存在」

**测试场景5：错误用户名或密码**
- 操作：输入正确的租户账号，错误的用户名或密码
- 预期结果：提示「用户名或密码错误」

## 2. 首页测试

**测试步骤**：
1. 成功登录系统后，进入首页
2. 查看系统概览数据
3. 检查导航菜单是否完整
4. 点击各导航菜单，验证跳转是否正常

**预期结果**：
- 首页加载正常，显示关键指标和数据统计
- 导航菜单完整，包含所有功能模块
- 点击导航菜单项，能正确跳转到对应模块页面

## 3. 商品管理模块测试

### 3.1 商品类别管理

#### 3.1.1 创建商品类别

**测试步骤**：
1. 从左侧导航栏，点击「商品管理」→「商品类别」
2. 点击页面右上角的「新增」按钮
3. 在弹出的表单中，填写以下信息：
   - 分类编码：M001
   - 分类名称：测试原材料
   - 上级分类：无（选择顶级分类）
   - 备注：测试分类
4. 点击「保存」按钮

**字段说明**：
- 分类编码：商品类别的唯一标识，必填，建议使用字母+数字组合
- 分类名称：商品类别的名称，必填
- 上级分类：选择父级分类，可选，顶级分类无需选择
- 备注：类别的描述信息，可选

**预期结果**：
- 商品类别创建成功，显示在类别列表中
- 类别列表能正确展示层级关系

#### 3.1.2 修改商品类别

**测试步骤**：
1. 在商品类别列表中，找到刚创建的「测试原材料」类别
2. 点击操作列的「编辑」按钮
3. 修改分类名称为：生产原材料
4. 点击「保存」按钮

**预期结果**：
- 商品类别修改成功，列表中显示更新后的名称

#### 3.1.3 删除商品类别

**测试步骤**：
1. 在商品类别列表中，找到「生产原材料」类别
2. 点击操作列的「删除」按钮
3. 在确认弹窗中，点击「确定」按钮

**预期结果**：
- 商品类别删除成功，从列表中移除

## 4. 单据管理模块测试

### 4.1 采购管理

#### 4.1.1 采购订单创建

**测试步骤**：
1. 从左侧导航栏，点击「单据管理」→「采购管理」→「采购订单」
2. 点击页面右上角的「新增」按钮
3. 在订单头信息中填写：
   - 供应商：选择或新增一个供应商
   - 单据日期：默认当前日期（或选择其他日期）
   - 备注：测试采购订单
4. 在订单明细表格中，点击「新增行」按钮
5. 在新增行中，点击「条码」列的选择按钮，弹出商品选择窗口
6. 在商品选择窗口中，选择之前创建的「测试商品」
7. 系统自动填充商品的名称、规格、型号、颜色、品牌、单位等信息
8. 在「数量」列填写：50
9. 在「单价」列填写：10.00
10. 系统自动计算金额、税额和价税合计
11. 点击「保存」按钮

**字段说明**：
- 供应商：采购订单的供应商，必填
- 单据日期：采购订单的创建日期，必填
- 单据编号：自动生成，必填
- 关联请购单：可选，关联已有的请购单
- 关联订单：可选，以销定购场景下关联销售订单
- 优惠率：针对商品总金额的优惠比例，可选
- 付款优惠：针对商品总金额的优惠金额，可选
- 优惠后金额：自动计算，优惠后的总金额
- 结算账户：用于结算的账户，可选
- 支付订金：预先支付的订金金额，可选
- 条码：商品的条码，必填，通过选择商品自动填充
- 数量：采购数量，必填
- 单价：采购单价，可选
- 金额：自动计算，数量*单价
- 税率：税率，可选
- 税额：自动计算，金额*税率
- 价税合计：自动计算，金额+税额

**预期结果**：
- 采购订单创建成功，显示在订单列表中
- 订单状态为「待审核」

#### 4.1.2 采购订单审核

**测试步骤**：
1. 在采购订单列表中，找到刚创建的订单
2. 点击操作列的「编辑」按钮
3. 在确认弹窗中，点击「保存并审核」按钮

**预期结果**：
- 采购订单审核成功，状态变为「已审核」

#### 4.1.3 采购入库

**测试步骤**：
1. 从左侧导航栏，点击「单据管理」→「采购管理」→「采购入库」
2. 点击页面右上角的「新增」按钮
3. 选择「采购订单」作为入库来源

## 12. 常见问题排查

### 12.1 页面加载缓慢
- 检查网络连接是否正常
- 检查浏览器缓存，清除缓存后重新加载
- 查看浏览器控制台是否有错误信息

### 12.2 操作失败无提示
- 检查浏览器控制台是否有JavaScript错误
- 检查网络请求是否成功（使用F12开发者工具的Network面板）
- 检查系统日志，查看是否有服务器端错误

### 12.3 数据不一致
- 检查数据库连接是否正常
- 检查相关单据是否都已审核
- 检查数据关联关系是否正确

### 12.4 权限问题
- 检查当前用户的角色和权限设置
- 检查功能模块的权限配置
