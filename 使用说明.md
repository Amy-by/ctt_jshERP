# ERP系统用户使用手册

## 第一章 系统概述

### 1.1 系统简介

本ERP系统是一套功能完善的企业资源计划管理系统，涵盖了物料管理、单据管理、财务管理、报表查询和系统管理等核心业务模块。系统采用前后端分离架构，后端基于Spring Boot框架构建，前端采用Vue.js技术栈，支持多租户模式，适用于中小型企业的日常运营管理。

系统的核心价值在于帮助企业实现业务流程的数字化管理，通过规范化的数据录入和流程控制，提高企业运营效率，降低管理成本。系统支持多角色权限管理，不同角色的用户可以看到和操作不同的功能模块，确保数据安全和业务规范。系统提供了直观的图形化界面，用户无需专业培训即可快速上手使用，大大降低了系统推广的难度和成本。

系统的技术特点包括多个方面。首先，系统采用HTTPS加密传输协议，有效保障数据传输过程中的安全性，防止敏感数据被窃取或篡改。其次，系统基于Redis实现会话管理，支持分布式部署和负载均衡，能够应对高并发访问场景。再次，系统采用MyBatis-Plus作为持久层框架，通过对象关系映射简化了数据库操作，降低了开发难度。最后，系统集成了Swagger接口文档，开发者和管理员可以方便地了解和使用各个API接口。

### 1.2 系统功能模块

系统按照企业日常运营管理的实际需求，将功能划分为五个核心模块和两个基础模块。核心模块包括物料管理、单据管理、财务管理、报表查询和系统管理，这些模块相互协作，共同支撑企业的日常业务运营。

**物料管理模块**是企业资源管理的基础，主要负责管理企业生产经营活动中涉及的商品和物料信息。该模块包含商品类别管理和物料信息管理两个子模块，通过建立完善的物料分类体系和详细的物料档案，帮助企业准确掌握物料信息，优化采购计划，降低库存成本。物料管理与其他业务模块紧密关联，采购管理模块的采购单据会自动增加相应物料的库存，销售管理模块的销售单据会自动扣减相应物料的库存，财务管理模块会引用物料的价格信息进行成本核算。

**单据管理模块**是企业业务流转的核心，负责管理采购、销售、库存调拨等各类业务单据。单据管理采用流程化设计，每笔业务都需要经过制单、审核等环节，确保业务数据的准确性和可追溯性。系统支持多种单据类型，包括采购入库单、销售出库单、库存调拨单、盘盈盘亏单等，满足企业不同业务场景的需求。

**财务管理模块**负责企业的资金管理和财务核算，主要包括账户管理、收付款管理、应收应付管理等功能。该模块与单据管理模块紧密联动，每当产生采购或销售业务时，系统会自动生成相应的应收应付记录，帮助财务人员及时掌握企业的资金状况。

**报表查询模块**为企业管理层提供决策支持，通过汇总和分析业务数据，生成各类统计报表。系统预置了采购统计、销售统计、库存统计、财务收支等多种报表，用户还可以根据自身需求自定义查询条件，快速获取所需数据。

**系统管理模块**用于维护系统的正常运行，包括用户管理、角色权限管理、系统参数配置、操作日志查询等功能。该模块通常仅对系统管理员开放，普通用户无法访问。

### 1.3 系统架构

系统采用经典的三层架构设计，包括表现层、业务逻辑层和数据访问层，三个层次相互独立又紧密协作。表现层负责用户界面交互，通过RESTful API与后端进行数据通信，采用前后端分离的开发模式，前端代码独立部署，互不干扰。业务逻辑层处理核心业务规则和数据验证，是系统的核心所在，所有业务逻辑都在这一层实现。数据访问层通过MyBatis-Plus框架与MySQL数据库进行交互，采用面向对象的方式操作数据库，大大简化了数据持久化操作的复杂度。

系统支持多租户隔离，每个租户拥有独立的数据空间，通过tenantId字段进行数据隔离。这种设计使得系统能够同时为多个企业提供服务，每个企业的数据都相互独立、安全可靠。系统管理员可以管理所有租户的数据，而普通用户只能访问其所属租户的数据，确保了数据的安全性和隔离性。

系统的部署架构支持水平扩展，可以通过增加服务实例来应对更高的并发访问需求。前端静态资源通过CDN分发，提高页面加载速度；后端服务支持集群部署，通过负载均衡器分发请求，实现请求的负载均衡和故障转移，提高系统的可用性和稳定性。

### 1.4 运行环境要求

后端服务需要Java 8或更高版本运行环境，建议内存配置不低于4GB，磁盘空间不低于50GB。数据库采用MySQL 5.7或MySQL 8.0，需要预先创建数据库并配置相应的连接信息，包括数据库名称、用户名、密码等。Redis缓存服务用于存储会话信息和热点数据，建议配置为主从复制模式以保证高可用性，同时需要设置合理的过期时间和内存限制。

前端服务需要Node.js 14.x或更高版本，通过npm或yarn包管理器安装依赖。开发环境下可以直接运行npm run dev命令启动开发服务器，生产环境建议使用Nginx作为Web服务器，配置HTTPS证书和静态资源缓存策略，以提高页面加载速度和安全性。

系统默认配置下，后端服务监听8443端口（HTTPS）和8080端口（HTTP），前端服务默认运行在3000端口。请确保这些端口在防火墙中已开放，并且没有其他服务占用。如果需要修改端口配置，可以修改application.yml配置文件中的相应配置项。

## 第二章 系统登录

### 2.1 访问系统

打开Chrome、Firefox或Edge等现代浏览器，在地址栏输入系统的前端访问地址，进入系统登录页面。建议使用最新版本的浏览器访问系统，以获得最佳的用户体验和安全性保障。如果页面无法正常显示，请按以下步骤排查问题：首先确认网络连接正常，能够访问服务器地址；其次检查浏览器是否支持JavaScript，建议启用Cookie和本地存储功能；最后确认系统服务已经启动运行，可以通过直接访问后端API地址来验证服务是否正常。

![系统登录页面](图片路径：images/login-page.png)

*图2-1：系统登录页面*

登录页面采用简洁的设计风格，页面中央为登录表单，包含用户名输入框、密码输入框和登录按钮。页面顶部显示系统名称和版本信息，底部显示技术支持联系方式和版权声明。页面背景采用渐变色设计，视觉体验舒适大方。

### 2.2 用户登录

在登录页面，按照以下步骤完成登录操作：

**第一步：在用户名输入框中输入您的登录账号。** 系统默认的管理员账号为"jsh"，普通用户账号由系统管理员分配。如果忘记用户名，请联系系统管理员获取。用户名不支持特殊字符，长度限制为3-20个字符。

![输入用户名](图片路径：images/login-username.png)

*图2-2：在用户名输入框中输入登录账号*

**第二步：在密码输入框中输入与账号对应的密码。** 系统默认密码为"123456"，首次登录后建议立即修改为更安全的密码。密码输入框支持显示隐藏切换，点击输入框右侧的眼睛图标可以切换密码的显示状态，方便用户确认输入是否正确。

![输入密码](图片路径：images/login-password.png)

*图2-3：在密码输入框中输入密码*

**第三步：点击"登录"按钮提交登录请求。** 系统会验证用户名和密码的正确性，验证通过后将自动跳转到系统首页。如果用户名或密码错误，页面会显示相应的错误提示信息，提示用户重新输入。

![点击登录按钮](图片路径：images/login-button.png)

*图2-4：点击登录按钮提交登录请求*

### 2.3 登录失败处理

如果登录失败，页面会显示具体的错误信息。常见的错误信息及处理方法如下：

**错误信息：用户不存在。** 请检查用户名输入是否正确，注意区分大小写。如果确认用户名正确但仍然提示用户不存在，可能是账号已被禁用或删除，请联系系统管理员确认账号状态。系统默认尝试登录次数为5次，超过限制后账号会被临时锁定15分钟。

**错误信息：密码错误。** 请确认密码输入是否正确，注意检查大小写和特殊字符。如果多次密码错误，账号可能会被临时锁定，请等待15分钟后重试或联系系统管理员解锁。系统会记录每次登录失败的IP地址和账号信息，用于安全审计。

**错误信息：验证码错误。** 如果系统启用了验证码功能，需要正确输入图片中显示的验证码字符。验证码不区分大小写，但需要注意区分外观相似的字符，如数字0和字母O、数字1和字母l、数字8和字母B等。如果看不清验证码，可以点击验证码图片刷新重新获取。

**错误信息：账号已过期。** 如果系统设置了账号有效期，账号过期后将无法登录。请联系系统管理员续期或重新开通账号。企业版用户可以联系技术支持进行处理。

### 2.4 修改初始密码

首次登录系统后，建议立即修改初始密码，以提高账户安全性。修改密码的步骤如下：

**第一步：点击页面右上角的用户名，在下拉菜单中选择"个人设置"选项。** 点击用户名后会展开一个下拉菜单，菜单中包含"个人设置"、"消息通知"和"退出登录"三个选项。

![进入个人设置](图片路径：images/user-menu.png)

*图2-5：点击用户名选择个人设置*

**第二步：在个人设置页面中，找到"修改密码"模块。** 页面会显示当前用户的基本信息，包括用户名、姓名、邮箱、手机号等。在页面下方找到"修改密码"区域，分别输入当前密码、新密码和确认密码。

![修改密码表单](图片路径：images/change-password.png)

*图2-6：填写修改密码表单*

**第三步：设置新密码。** 新密码需要满足以下复杂度要求：长度不少于8个字符，包含大写字母、小写字母、数字和特殊字符中的至少三种类型。新密码不能与最近使用过的5个密码相同。

| 密码要求 | 说明 |
|---------|------|
| 最小长度 | 8个字符 |
| 字符类型 | 至少包含大写字母、小写字母、数字中的两种 |
| 特殊字符 | 建议包含至少一个特殊字符 |
| 历史检查 | 不能与最近5个密码相同 |

**第四步：点击"保存"按钮提交修改请求。** 系统会验证当前密码的正确性，并检查新密码的复杂度是否符合要求。修改成功后，系统会提示密码修改成功，下次登录时需要使用新密码。

![密码修改成功](图片路径：images/password-success.png)

*图2-7：密码修改成功*

## 第三章 系统主界面

### 3.1 界面布局概览

成功登录系统后，将进入系统主界面。主界面采用标准的Web应用布局设计，由顶部导航栏、左侧菜单栏和主内容区域三部分组成。这种布局方式充分利用了屏幕空间，同时保持了界面的简洁和美观。

![系统主界面](图片路径：images/main-interface.png)

*图3-1：系统主界面布局*

**顶部导航栏**位于页面最上方，包含系统logo、当前用户信息、功能菜单和退出登录按钮。导航栏高度为60像素，采用固定定位方式，始终显示在页面顶部，即使页面内容较长也不会消失。

**左侧菜单栏**采用折叠式设计，默认显示一级功能菜单，点击菜单项可以展开显示二级功能。菜单栏宽度为200像素，支持拖拽调整宽度，可以根据实际需要展开或折叠。

**主内容区域**占据页面的大部分空间，用于显示具体的功能操作界面。内容区域采用响应式设计，能够根据浏览器窗口大小自动调整布局，确保在各种分辨率下都能获得良好的显示效果。

### 3.2 顶部导航栏

顶部导航栏提供了系统级的功能入口和用户信息展示。从左到右依次包含以下元素：

**系统Logo区域**：显示系统名称和图标，点击可以返回首页。Logo旁边显示当前系统的版本号，方便用户了解系统版本信息。Logo区域支持点击，点击后系统会自动跳转到首页。

**用户信息区域**：显示当前登录用户的头像（如果有）和用户名。点击用户名会展开一个下拉菜单，包含"个人设置"、"消息通知"和"退出登录"三个选项。

## 第四章 物料管理

### 4.1 功能概述

物料管理是ERP系统的基础模块之一，主要用于管理企业生产经营活动中涉及的商品和物料信息。良好的物料管理能够帮助企业准确掌握库存状况，优化采购计划，降低库存成本。本模块包含以下核心功能：商品类别管理，用于建立多级分类体系，便于物料的分类检索；物料基本信息管理，包括物料的名称、规格、单位、参考价格等信息的维护；物料库存查询，用于查看各物料的当前库存数量和库存变动历史。

物料管理与其他模块紧密关联。采购管理模块的采购单据会自动增加相应物料的库存；销售管理模块的销售单据会自动扣减相应物料的库存；财务管理模块会引用物料的价格信息进行成本核算。因此，物料基础数据的准确性对整个系统的数据质量至关重要。建议企业建立物料编码规范，统一物料的命名和分类标准，避免出现物料重复或信息混乱的情况。

### 4.2 商品类别管理

商品类别采用树形结构设计，支持多级分类，便于对物料进行层级化管理。例如，可以建立"电子产品"一级分类，其下设置"电脑设备"、"外设配件"等二级分类，进一步细分可以设置"笔记本电脑"、"台式电脑"、"平板电脑"等三级分类。合理的分类体系能够帮助企业建立清晰的物料结构，便于后续的统计分析和决策支持。

## 第五章 单据管理

### 5.1 功能概述

单据管理是企业业务流转的核心模块，负责管理采购、销售、库存调拨等各类业务单据。单据管理采用流程化设计，每笔业务都需要经过制单、审核等环节，确保业务数据的准确性和可追溯性。系统支持多种单据类型，包括采购入库单、销售出库单、库存调拨单、盘盈盘亏单等，满足企业不同业务场景的需求。

单据管理模块的核心价值在于规范化业务流程，确保每一笔业务都有据可查。通过单据的状态流转，系统能够追踪业务的完整生命周期，从创建、审核、执行到完成，每个环节都有详细的时间戳和操作人记录。这种设计不仅提高了数据的准确性，也为企业的内部审计和风险控制提供了有力支持。

### 5.2 采购入库单

采购入库单用于记录企业采购物料的入库业务，是增加库存的主要单据之一。当企业与供应商完成采购后，需要创建采购入库单，将物料实际入库，同时更新库存数量。

#### 5.2.1 创建采购入库单

**第一步：在左侧菜单栏中依次点击"单据管理" → "采购入库"。** 进入采购入库单列表页面，页面显示所有已创建的采购入库单列表。

![进入采购入库管理](图片路径：images/purchase-inbound.png)

*图5-1：进入采购入库管理*

**第二步：点击工具栏中的"新增"按钮。** 弹出新增采购入库单对话框，页面分为基本信息区和明细区两部分。

![新增采购入库单](图片路径：images/purchase-inbound-add.png)

*图5-2：新增采购入库单*

**第三步：填写基本信息。** 必填项包括供应商、入库仓库、业务日期，可选项包括经手人、单据编号、备注说明。

| 字段名称 | 是否必填 | 说明 |
|---------|---------|------|
| 供应商 | 是 | 选择本次采购的供应商，可通过下拉框搜索选择 |
| 入库仓库 | 是 | 选择物料入库的仓库，物料将增加该仓库的库存 |
| 业务日期 | 是 | 选择单据的业务日期，通常为实际入库日期 |
| 经手人 | 否 | 选择业务经手人，默认为当前登录用户 |
| 单据编号 | 否 | 系统自动生成，也可手动输入自定义编号 |
| 备注说明 | 否 | 填写业务备注信息 |

**第四步：添加入库明细。** 点击"添加物料"按钮，弹出物料选择对话框。在对话框中选择需要入库的物料，输入入库数量和采购单价。

![添加入库明细](图片路径：images/purchase-inbound-items.png)

*图5-3：添加入库明细*

**第五步：确认单据信息无误后，点击"保存并提交审核"按钮。** 系统会验证单据信息的完整性，验证通过后保存单据。单据默认状态为"待审核"，需要审核通过后库存才会实际增加。

### 5.3 销售出库单

销售出库单用于记录企业销售物料的出库业务，是减少库存的主要单据之一。当企业与客户完成销售后，需要创建销售出库单，将物料实际出库，同时扣减库存数量。

#### 5.3.1 创建销售出库单

**第一步：在左侧菜单栏中依次点击"单据管理" → "销售出库"。** 进入销售出库单列表页面。

**第二步：点击工具栏中的"新增"按钮。** 弹出新增销售出库单对话框。

![新增销售出库单](图片路径：images/sales-outbound-add.png)

*图5-4：新增销售出库单*

**第三步：填写基本信息。** 必填项包括客户、出库仓库、业务日期，可选项包括经手人、单据编号、备注说明。

**第四步：添加出库明细。** 点击"添加物料"按钮，选择需要出库的物料，系统会自动显示该物料的当前库存数量和参考售价。输入销售数量和实际售价。

**第五步：确认信息后点击"保存并提交审核"按钮。** 单据保存后进入待审核状态，审核通过后库存会自动扣减。

### 5.4 单据审核

所有业务单据都需要经过审核确认后才能执行实际业务操作。审核流程确保了业务的规范性和数据的准确性。

**审核操作步骤：**

**第一步：在单据列表中找到需要审核的单据。** 单据状态为"待审核"时可以进行审核操作。点击单据行查看单据详情。

![待审核单据](图片路径：images/bill-pending.png)

*图5-5：待审核单据*

**第二步：审核人员核对单据信息。** 确认业务内容、金额、数量等信息无误后，点击工具栏中的"审核"按钮。

![审核按钮](图片路径：images/audit-button.png)

*图5-6：审核按钮*

**第三步：系统提示审核成功。** 单据状态变更为"已审核"，同时系统自动执行相应的库存操作（如入库或出库）。

**审核不通过处理：** 如果发现单据存在问题，可以点击"驳回"按钮，填写驳回原因后将单据退回给制单人修改。驳回后的单据状态为"已驳回"，制单人可以在修改后重新提交审核。

## 第六章 财务管理

### 6.1 功能概述

财务管理模块负责企业的资金管理和财务核算，主要包括账户管理、收付款管理、应收应付管理等功能。该模块与单据管理模块紧密联动，每当产生采购或销售业务时，系统会自动生成相应的应收应付记录，帮助财务人员及时掌握企业的资金状况。

财务管理模块采用双轨制设计，既支持与业务单据自动关联，也支持手工录入财务数据。这种设计使得系统能够适应不同企业的管理需求，既可以实现业务流程与财务流程的一体化管理，也可以在业务数据不完整时进行手工补充。

### 6.2 账户管理

账户管理用于维护企业的银行账户和现金账户信息，是收付款操作的基础数据。在进行收付款操作前，需要先在系统中创建相应的账户信息。

| 账户类型 | 是 | 选择账户类型，包括银行账户、现金账户、第三方支付等 |

| 账号 | 否 | 银行账户必填，输入银行账号 |

## 第七章 报表查询

### 7.1 功能概述

报表查询模块为企业管理层提供决策支持，通过汇总和分析业务数据，生成各类统计报表。系统预置了采购统计、销售统计、库存统计、财务收支等多种报表，用户还可以根据自身需求自定义查询条件，快速获取所需数据。

报表查询模块采用灵活的设计，支持多维度的数据分析和展示。用户可以通过时间范围、业务类型、客户供应商、仓库等多个维度筛选数据，报表结果支持表格展示和图表展示两种形式，满足不同场景的分析需求。

### 7.2 采购统计报表

采购统计报表用于汇总企业的采购业务数据，帮助管理层了解采购支出情况和供应商表现。

**查询步骤：**

## 第九章 常见问题与解决方案

### 9.1 登录相关问题

**问题一：忘记密码怎么办？**

如果忘记了登录密码，可以通过以下方式重置密码：

1. 联系系统管理员，说明情况并申请重置密码
2. 管理员登录系统后，在用户管理中找到对应用户
3. 点击"重置密码"按钮，将密码重置为初始密码
4. 重置后首次登录时，系统会强制要求修改密码

**问题二：账号被锁定怎么办？**

系统默认在连续5次密码错误后锁定账号15分钟。锁定期间无法登录，可以采取以下措施：

1. 等待15分钟后自动解锁
2. 联系系统管理员立即解锁
3. 如果多次被锁定，建议检查是否账号信息泄露

### 9.2 业务操作问题

**问题三：创建单据时找不到物料怎么办？**

如果在创建单据时无法找到需要的物料，请按以下步骤排查：

1. 确认物料是否已经创建：在"物料管理"中搜索物料名称或编号
2. 确认物料是否关联了正确的分类：物料需要归属于某个商品分类才能被检索
3. 确认用户是否有查看该物料的权限：联系管理员检查角色权限设置
4. 确认物料是否被标记为停用：停用的物料不会显示在选择列表中

**问题四：单据审核不通过怎么办？**

如果提交的单据被审核人员驳回，请按以下步骤处理：

1. 在单据列表中找到被驳回的单据，查看驳回原因
2. 点击单据进入详情页面，修改单据内容
3. 修改完成后重新提交审核
4. 如有疑问，可以联系审核人员沟通确认

### 9.3 系统技术问题

**问题五：页面加载缓慢怎么办？**

如果系统页面加载速度较慢，可以尝试以下优化措施：

1. 检查网络连接是否稳定
2. 清除浏览器缓存和Cookie
3. 尝试使用其他浏览器（如Chrome、Firefox）
4. 减少同时打开的浏览器标签页数量
5. 联系管理员检查服务器性能和网络状况

**问题六：数据导出失败怎么办？**

如果导出报表时出现失败，请按以下步骤排查：

1. 检查导出数据量是否过大：大量数据导出可能需要较长时间
2. 确认浏览器是否允许下载文件：检查浏览器的弹出窗口拦截设置
3. 尝试分批次导出：将数据分成小批次分别导出
4. 更换浏览器或电脑后重试
